QUploader Documentation

QUploader is a highly flexible, responsive, and customizable jQuery file upload plugin. It provides out-of-the-box support for Drag & Drop, WebRTC Camera captures, Folder Structure uploading with depth limits, Image Auto-Resizing, Base64 encoding, Chunked Resumable uploads, and an innovative Headless API for custom front-end bindings.

Built with TypeScript and packaged with Vite, it delivers state-of-the-art uploading capabilities with zero bloated external styling dependencies.

Quick Start

To initialize QUploader, include jQuery and the QUploader script/CSS in your document. Target a container or a file input element and invoke the plugin method:

HTML Structure

<!-- Standard container initialization -->
<div id="uploader"></div>

<!-- Or initialize on a file input directly -->
<input type="file" id="file-uploader" multiple accept="image/*" />

Javascript Initialization

import $ from 'jquery';
import '/src/quploader.ts'; // ESM Import

$(document).ready(function() {
  $('#uploader').quploader({
    uploadUrl: '/api/upload',
    multiple: true,
    dragDrop: true,
    maxFileSize: '10MB',
    fileTypes: ['.jpg', '.png', '.pdf'],
    onSuccess: function(file, response) {
      console.log('Successfully uploaded:', file.name);
    }
  });
});

Configuration Options

Customize the uploader behavior by passing an object containing any of the options below during initialization:

Option Type / Default Description
uploadUrl
optional 		string
'' 		The API endpoint url where files will be uploaded via POST request. Can be ignored in headless/no-upload flows.
chunkUploadUrl
optional 		string
undefined 		API endpoint specifically dedicated to handling chunk/fragment uploads (used when chunking is triggered).
multiple boolean
false 		Whether to allow picking and uploading multiple files concurrently.
dragDrop boolean
true 		Enables drag-and-drop zone events on the uploader container.
browseButton boolean
true 		Specifies if the default "Browse File" button is generated in the UI.
clickReviewAreaToBrowse boolean
false 		If true, clicking inside the file review list triggers the file browse dialog.
cameraButton boolean
false 		Generates a "Camera" trigger button enabling capturing snapshots directly using the user's WebRTC device.
showIntroText boolean
true 		Displays default drag-drop instructions inside the container.
allowFolder boolean
false 		Enables choosing and reading complete nested folder directory structures.
subfolderLevel number
undefined 		Sets the maximum nesting directory depth allowed when reading folders (e.g. 0 only reads root files inside the folder).
fileTypes string[]
[] 		Array of accepted file extensions or mime categories, e.g. ['.png', '.jpeg', 'image/*'].
maxFileSize number | string
10MB 		Maximum file size limit. Supports bytes (number) or strings like '5MB', '500KB'.
chunkSize number | string
undefined 		If configured, splits files exceeding this size into chunk segments uploaded individually (e.g. '2MB').
retry number
3 		Maximum retry attempts per chunk upload segment in case of server timeouts or network failures.
deleteUrl string
undefined 		API endpoint invoked via POST containing { key: serverKey } when a successfully uploaded file is deleted.
reviewMode string
'thumbnail' 		UI Layout mode for file list: 'thumbnail' (grid layout), 'detail' (detailed linear list), or 'single' (fills container, ideal for single avatar/image uploads).
reviewPosition string
'below' 		Position of the preview list container relative to the upload trigger area: 'below', 'above', or 'none'.
errorDelay number
3000 		Time in milliseconds to show local validation errors before auto-fading them.
resize object
undefined 		Object structure: { maxWidth: 800, maxHeight: 600 }. Client-side downsizes image files prior to transmission.
autoUpload boolean
true 		If true, begins transferring files to server automatically immediately after they are queued.
uploadAsBase64 boolean
false 		If set to true, posts files as JSON-payload base64 data structures rather than traditional multi-part Form Data.
darkMode boolean | 'auto'
false 		Enables the built-in modern dark-mode stylesheet palette. Setting to 'auto' matches system dark queries.
resumable boolean
true 		Enables saving chunk upload markers to localStorage to allow resuming partially finished files on reconnect.
preferFileInput boolean
true 		Uses standard hidden input clicks to browse files. If set to false, leverages the modern File System Access API where supported.

API Methods

Methods are called imperatively on an already initialized QUploader instance by passing the method name as a string to the jQuery selector:

uploadAll

Triggers/forces uploading of all currently queued files having pending or error statuses.

$('#uploader').quploader('uploadAll');

clearQueue

Clears the entire file queue, cancels pending XMLHttpRequests, and sends server deletion requests for successfully uploaded items.

$('#uploader').quploader('clearQueue');

getFiles

Returns an array containing all files currently managed by the uploader context. Each file includes properties like status, progress, and custom helper methods.

const fileList = $('#uploader').quploader('getFiles');
console.log(fileList);

browseFile

Triggers the file browse picker dialog. Highly useful for Headless mode.

$('#myButton').quploader('browseFile');

browseFolder

Triggers the folder selection picker dialog.

$('#myButton').quploader('browseFolder');

camera

Triggers the browser WebRTC modal dialog permitting immediate snap captures.

$('#myButton').quploader('camera');

Events & Callbacks

Capture lifecycles and hook details by defining callback functions during initialization:

onPick(files)

Executed immediately when files are selected by the user, before validation and UI rendering. Received parameter is a native File[] list.

$('#uploader').quploader({
  onPick: function(files) {
    console.log("Picked files count:", files.length);
  }
});

onValidate(file)

A custom validator hook. Returning false rejects the file from entering the upload queue.

$('#uploader').quploader({
  onValidate: function(file) {
    if (file.name.includes('restricted')) {
      alert('This file is restricted.');
      return false;
    }
    return true;
  }
});

onUploadStart(file)

Fires when a file begins the HTTP upload process.

$('#uploader').quploader({
  onUploadStart: function(file) {
    console.log("Uploading file:", file.name);
  }
});

onProgress(file, progress)

Triggered continuously during transfer. The progress parameter is a rounded integer from 0 to 100.

$('#uploader').quploader({
  onProgress: function(file, progress) {
    console.log(`Progress for ${file.name}: ${progress}%`);
  }
});

onSuccess(file, response)

Invoked when the server returns a successful response code (2xx). response is the parsed JSON/string payload from the server.

$('#uploader').quploader({
  onSuccess: function(file, response) {
    console.log("Server response:", response);
  }
});

onError(file, error)

Triggers if a file fails client-side limits, local validator checks, or encounters network/upload errors.

$('#uploader').quploader({
  onError: function(file, error) {
    console.error(`Error on file ${file.name}: ${error}`);
  }
});

onDeleted(file)

Triggered when a file is successfully deleted from the local client-side queue list.

$('#uploader').quploader({
  onDeleted: function(file) {
    console.log("Removed file:", file.name);
  }
});

Headless Mode

Headless Mode allows you to tie file pickers, camera triggers, drag-and-drop zones, or folder selections to any arbitrary HTML element of your choice without loading the default uploader containers, file lists, or status bars.

This allows full control over the UI, letting you build custom file grids, overlays, or upload lists while delegating selection, validation, and chunked uploads to QUploader.

Initializing Headless Triggers

To initialize elements in headless mode, call quploader by passing the trigger mode name as the first argument, and configuring options in the second argument. headless: true is applied automatically.

// Bind browse file picker to a custom button
$('#myButton').quploader('browseFile', {
  autoUpload: false,
  onPick: function(files) {
    console.log("Files chosen:", files);
  }
});

// Turn a custom div into a drag and drop zone
$('#myDropArea').quploader('dropzone', {
  autoUpload: false,
  onPick: function(files) {
    console.log("Files dropped:", files);
  }
});
